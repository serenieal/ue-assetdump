## 2.3 검증 운영 원칙
- **공식 검증 기준은 UE 내부 수동 덤프 결과(JSON)로 둔다.**
- 외부 자동 실행 레이어(MCP/adapter/runner)는 **보조 참고용**으로만 사용한다.
- 외부 실행 결과가 수동 덤프 결과와 다르면 **수동 덤프 결과를 우선**한다.
- 단계 게이트 승인(Gate B ~ Gate G)은 가능한 한 **수동 덤프 결과 기준**으로 판정한다.
- 외부 실행은 빠른 가설 검증/로그 확인용으로만 쓰고, 완료 판정의 단독 근거로 사용하지 않는다.
- 타임아웃 또는 외부 실행 불안정이 계속되면, 해당 문제는 별도 보조 이슈로 분리하고 본 개발 흐름은 수동 덤프 기준으로 계속 진행한다.
- **수동 덤프 경로가 완전해지면 외부 검증 인프라는 종료 대상으로 본다.**
- 따라서 레거시 commandlet/외부 runner 검증이 일부 불완전하더라도, **수동 덤프 기준 기능과 단계 목표가 충족되면 해당 단계는 통과 처리할 수 있다.**
